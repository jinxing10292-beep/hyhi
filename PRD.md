### 1.1 제품 소개

**검 강화 머지 게임**은 HTML5 기반의 웹 브라우저 게임으로, 검을 합성(머지)하고 강화하여 골드를 획득하는 중독성 있는 방치형 게임입니다. 사용자는 같은 레벨의 검 2개를 합쳐 더 높은 레벨의 검을 만들고, 강화 시스템을 통해 검의 성능을 향상시킬 수 있습니다.

### 1.2 제품 배경

- **시장 분석**: 머지 게임 장르는 모바일 게임 시장에서 높은 인기를 보이고 있으며, 특히 캐주얼 게이머들에게 큰 호응을 얻고 있습니다.
- **차별화 포인트**: 
  - 웹 브라우저 기반으로 별도 설치 없이 즉시 플레이 가능
  - 직관적인 드래그 앤 드롭 인터페이스
  - 강화 시스템의 긴장감과 보상 시스템의 결합
  - 로컬 스토리지 기반 자동 저장으로 진행상황 보존

### 1.3 핵심 가치 제안

1. **접근성**: 웹 브라우저만 있으면 언제 어디서나 플레이 가능
2. **중독성**: 지수적으로 증가하는 보상 시스템
3. **전략성**: 합성 타이밍과 강화 선택의 전략적 요소
4. **성취감**: 퀘스트, 업적 시스템을 통한 지속적인 목표 제공

## 2. 제품 비전 및 목표

### 2.1 비전

"누구나 쉽게 즐길 수 있는 중독성 있는 웹 기반 머지 게임을 만들어, 짧은 시간에도 성취감을 느낄 수 있는 경험을 제공한다."

### 2.2 비즈니스 목표

- **단기 목표 (1-3개월)**
  - 일일 활성 사용자(DAU) 1,000명 달성
  - 평균 세션 시간 10분 이상 유지
  - 사용자 리텐션율 30% 이상 (7일 기준)

- **중기 목표 (3-6개월)**
  - 월간 활성 사용자(MAU) 10,000명 달성
  - 평균 세션 시간 15분 이상 유지
  - 사용자 리텐션율 40% 이상 (30일 기준)

- **장기 목표 (6-12개월)**
  - MAU 50,000명 달성
  - 수익화 모델 도입 (광고, 프리미엄 기능)
  - 모바일 앱 버전 출시

### 2.3 성공 지표 (KPI)

1. **사용자 참여도**
   - DAU/MAU 비율: 30% 이상
   - 평균 세션 시간: 10분 이상
   - 세션당 평균 액션 수: 50회 이상

2. **사용자 유지율**
   - 1일 리텐션: 50% 이상
   - 7일 리텐션: 30% 이상
   - 30일 리텐션: 15% 이상

3. **게임 진행도**
   - 평균 최고 검 레벨: Lv.5 이상
   - 퀘스트 완료율: 60% 이상
   - 업적 달성률: 40% 이상

## 3. 타겟 사용자

### 3.1 주요 페르소나

**페르소나 1: 캐주얼 게이머 "민수" (25세, 직장인)**
- **특징**: 출퇴근 시간이나 점심시간에 짧게 즐길 게임을 찾음
- **니즈**: 복잡하지 않고 직관적인 게임, 짧은 시간에도 성취감
- **플레이 패턴**: 하루 3-5회, 회당 5-10분
- **목표**: 스트레스 해소, 짧은 여가 시간 활용

**페르소나 2: 방치형 게임 애호가 "지은" (22세, 대학생)**
- **특징**: 방치형 게임을 즐기며 효율적인 전략 수립을 좋아함
- **니즈**: 자동 진행 시스템, 최적화된 전략 수립 가능
- **플레이 패턴**: 하루 10회 이상, 회당 2-3분 (체크)
- **목표**: 최고 레벨 달성, 모든 업적 완료

**페르소나 3: 컬렉터 "현우" (30세, 프리랜서)**
- **특징**: 게임 내 모든 요소를 수집하고 완성하는 것을 즐김
- **니즈**: 다양한 업적, 수집 요소, 진행도 추적
- **플레이 패턴**: 하루 5-7회, 회당 10-15분
- **목표**: 모든 업적 달성, 최고 레벨 검 보유

### 3.2 사용자 세그먼트

1. **라이트 유저 (40%)**
   - 주 1-3회 플레이
   - 기본 기능만 사용
   - 목표: 간단한 재미

2. **미디엄 유저 (40%)**
   - 주 4-7회 플레이
   - 대부분의 기능 활용
   - 목표: 꾸준한 진행

3. **헤비 유저 (20%)**
   - 매일 여러 번 플레이
   - 모든 기능 마스터
   - 목표: 최고 레벨 달성, 완벽한 플레이

## 4. 핵심 기능 명세

### 4.1 인벤토리 시스템

#### 4.1.1 기본 사양

- **크기**: 5x5 그리드 (총 25칸)
- **슬롯 타입**: 동일한 크기의 정사각형 슬롯
- **시각적 피드백**:
  - 빈 슬롯: 어두운 반투명 배경 (#1e283e, 80% opacity)
  - 검이 있는 슬롯: 그라데이션 배경 (#4a5580 → #5a6590)
  - 호버 상태: 파란색 테두리 (#1e90ff) + 글로우 효과
  - 드래그 중: 투명도 30%, 약간 축소
  - 드롭 가능 영역: 노란색 테두리 (#ffc107) + 강한 글로우

#### 4.1.2 드래그 앤 드롭 메커니즘

**요구사항:**
- 검이 있는 슬롯만 드래그 가능
- 드래그 시작 시 투명한 드래그 이미지 사용 (깜빡임 방지)
- 드래그 중인 슬롯은 시각적으로 표시 (opacity 0.3)
- 드롭 가능한 슬롯 위에 마우스 오버 시 하이라이트
- 드롭 시 즉시 합성 또는 위치 교환 처리

**합성 조건:**
- 같은 레벨의 검 2개
- 같은 강화 단계 (enhancement level)
- 조건 충족 시 자동으로 다음 레벨 검 생성

**위치 교환:**
- 합성 조건 미충족 시 두 슬롯의 검 위치 교환
- 빈 슬롯으로 드래그 시 해당 위치로 이동

#### 4.1.3 검 정보 표시

각 슬롯에 표시되는 정보:
1. **검 아이콘**: ⚔️ 이모지 (2.5em 크기)
2. **레벨 표시**: "Lv.X" 형식
   - Lv.1-2: 노란색 (#ffc107)
   - Lv.3-4: 파란색 (#2196f3)
   - Lv.5-6: 보라색 (#9c27b0)
   - Lv.7-9: 핑크색 (#e91e63)
   - Lv.10+: 빨간색 (#ff1744)
3. **강화 단계**: "+X" 형식 (강화된 경우만, 초록색 #4caf50)
4. **초당 골드**: "X.XXG/s" 형식 (회색 #aaa)

### 4.2 검 합성 시스템

#### 4.2.1 합성 메커니즘

**프로세스:**
1. 사용자가 검을 드래그하여 같은 레벨의 다른 검 위에 드롭
2. 시스템이 합성 조건 검증
3. 조건 충족 시:
   - 원본 슬롯 비우기
   - 대상 슬롯에 (레벨 + 1) 검 생성
   - 강화 단계는 0으로 초기화
   - 합성 카운터 증가
   - 플로팅 텍스트로 "🗡️ Lv.X 검 획득!" 표시
4. 퀘스트 및 업적 진행도 업데이트

#### 4.2.2 레벨별 골드 획득량

**공식**: `0.1 × 10000^((level-1)/9)`

**레벨별 상세 수치:**


| 레벨 | 초당 골드 | 배율 (이전 레벨 대비) | 시간당 골드 |
|------|-----------|---------------------|------------|
| Lv.1 | 0.1 G/s | - | 360 G/h |
| Lv.2 | 0.46 G/s | 4.64x | 1,670 G/h |
| Lv.3 | 2.15 G/s | 4.64x | 7,740 G/h |
| Lv.4 | 10.0 G/s | 4.64x | 36,000 G/h |
| Lv.5 | 46.4 G/s | 4.64x | 167,040 G/h |
| Lv.6 | 215 G/s | 4.64x | 774,000 G/h |
| Lv.7 | 1,000 G/s | 4.64x | 3,600,000 G/h |
| Lv.8 | 4,641 G/s | 4.64x | 16,707,600 G/h |
| Lv.9 | 21,544 G/s | 4.64x | 77,558,400 G/h |
| Lv.10 | 100,000 G/s | 4.64x | 360,000,000 G/h |

**설계 의도:**
- 초기 레벨: 느린 진행으로 게임 메커니즘 학습
- 중간 레벨 (Lv.4-6): 가속화된 진행으로 성취감 제공
- 고레벨 (Lv.7+): 폭발적인 성장으로 장기 목표 제공

### 4.3 검 강화 시스템

#### 4.3.1 강화 메커니즘

**프로세스:**
1. 사용자가 인벤토리에서 검 클릭하여 선택
2. "강화" 버튼 클릭
3. 강화 모달 창 표시:
   - 현재 검 정보 (레벨, 강화 단계)
   - 강화 비용
   - 성공/유지/파괴 확률
   - 경고 메시지
4. 사용자가 "강화하기" 버튼 클릭
5. 랜덤 결과 생성 및 적용
6. 결과 플로팅 텍스트로 표시

#### 4.3.2 강화 확률 시스템

**공식:**
- 성공률: `max(30%, 70% - enhancement × 10%)`
- 유지율: `100% - 성공률 - 10%`
- 파괴율: `10%` (고정)

**강화 단계별 확률:**

| 강화 단계 | 성공률 | 유지율 | 파괴율 | 비용 |
|----------|--------|--------|--------|------|
| +0 → +1 | 70% | 20% | 10% | 50 G |
| +1 → +2 | 60% | 30% | 10% | 100 G |
| +2 → +3 | 50% | 40% | 10% | 200 G |
| +3 → +4 | 40% | 50% | 10% | 400 G |
| +4 → +5 | 30% | 60% | 10% | 800 G |
| +5 → +6 | 30% | 60% | 10% | 1,600 G |
| +6 → +7 | 30% | 60% | 10% | 3,200 G |

**강화 비용 공식**: `50 × 2^enhancement`

**강화 보너스:**
- 각 강화 단계마다 골드 획득량 +20%
- 공식: `base_gold × 1.2^enhancement`
- 예시: Lv.5 +3 검 = 46.4 × 1.2³ = 80.3 G/s

#### 4.3.3 강화 결과 처리

**성공:**
- 강화 단계 +1
- 총 강화 횟수 카운터 증가
- 플로팅 텍스트: "✨ 강화 성공! +X"
- 퀘스트/업적 업데이트

**유지:**
- 변화 없음
- 플로팅 텍스트: "😐 강화 유지"

**파괴:**
- 검 삭제 (인벤토리에서 제거)
- 플로팅 텍스트: "💥 검 파괴!"
- 선택 해제

### 4.4 상점 시스템

#### 4.4.1 검 구매

**기본 사양:**
- 가격: 20 골드 (고정)
- 획득: Lv.1 검 (강화 +0)
- 제한: 인벤토리 빈 공간 필요
- 버튼 위치: 하단 액션 바 좌측

**구매 프로세스:**
1. 골드 충분 여부 확인
2. 빈 슬롯 존재 여부 확인
3. 조건 충족 시:
   - 골드 차감
   - 첫 번째 빈 슬롯에 Lv.1 검 생성
   - 플로팅 텍스트: "⚔️ Lv.1 검 획득!"
4. 조건 미충족 시 에러 메시지 표시

#### 4.4.2 행운 상자

**기본 사양:**
- 가격: 100 골드
- 랜덤 보상 시스템
- 버튼 위치: 하단 액션 바 중앙

**보상 테이블:**

| 루비 3개 | 4% | 💎💎💎 루비 3개! |
| 골드 500 | 10% | 💰💰💰 골드 500! |

**특수 케이스:**
- 인벤토리 가득 + 검 보상: 골드 100 환불

**모달 UI:**
- 애니메이션: 상자 아이콘 (🎁) 바운스 효과

### 4.5 퀘스트 시스템

- 추가 보상 제공

- 실시간 업데이트

**모달 구성:**

  - 설명 (작은 글씨)
  - 진행도 (X / Y)
  - 보상 표시 (완료 시 "완료!", 미완료 시 "XG")
- 완료된 업적: 배경색 변경 (노란색 테두리)

**자동 보상:**
- 업적 달성 시 자동으로 골드 지급
- 플로팅 텍스트: "🏆 업적 달성! +XG"
- 배지 업데이트 (새로운 달성 가능 업적 수)

### 4.7 정렬 및 관리 기능

#### 4.7.1 자동 정렬

**정렬 기준:**
1. 1차: 레벨 (내림차순)
2. 2차: 강화 단계 (내림차순)

**동작:**
- "🔄 정렬 (레벨순)" 버튼 클릭
- 모든 검을 추출하여 정렬
- 인벤토리 초기화 후 순서대로 배치
- 플로팅 텍스트: "📊 정렬 완료!"

**사용 시나리오:**
- 인벤토리가 복잡해졌을 때
- 같은 레벨 검을 쉽게 찾고 싶을 때
- 고레벨 검을 한눈에 보고 싶을 때

#### 4.7.2 전체 판매

**기능:**
- 모든 검을 골드로 변환
- 버튼: "💰 전체 판매 (골드 획득)"

**가격 계산 공식:**
```
검 가격 = 10 × 2^(level-1) × (1 + enhancement × 0.5)
```

**예시:**
- Lv.1 +0: 10 G
- Lv.3 +0: 40 G
- Lv.5 +2: 320 × 2 = 640 G
- Lv.7 +0: 640 G

**확인 프로세스:**
1. 버튼 클릭
2. 총 판매 금액 계산
3. 확인 다이얼로그: "모든 검을 XG에 판매하시겠습니까?"
4. 확인 시:
   - 인벤토리 전체 비우기
   - 골드 지급
   - 플로팅 텍스트: "💰 XG 획득!"

### 8.2 중기 진행 (Lv.5-7)

**목표: 2-3시간 내 Lv.7 달성**

**특징:**
- 골드 획득 속도 급증
- 강화 시스템 활용 시작
- 행운 상자 구매 가능

**전략:**
- Lv.5 검 여러 개 보유
- 선택적 강화 (+1~+2)
- 퀘스트 완료로 추가 골드

### 8.3 후기 진행 (Lv.7-10)

**목표: 10시간 이상 플레이로 Lv.10 달성**

**특징:**
- 폭발적인 골드 증가
- 고위험 강화 시도
- 장기 목표 달성

**전략:**
- Lv.7+ 검 집중 육성
- 강화 +3 이상 도전
- 전체 판매 후 재투자

### 8.4 골드 인플레이션 관리

**문제:**
- 고레벨에서 골드가 너무 빠르게 증가
- 초기 가격(20G)이 무의미해짐

**해결책 (향후):**
- 프레스티지 시스템 도입
- 루비 기반 영구 업그레이드
- 새로운 골드 소비처 추가

### 8.5 강화 시스템 밸런스

**설계 의도:**
- 초기 강화 (+0~+2): 비교적 안전, 저렴
- 중기 강화 (+3~+4): 위험 증가, 보상 증가
- 후기 강화 (+5+): 고위험 고보상

**기대 가치 계산:**

| 강화 | 비용 | 성공 시 이득 | 기대값 | 추천 |

| +0→+1 | 50 G | +20% 골드 | 긍정적 | ✅ 추천 |
| +1→+2 | 100 G | +44% 골드 | 긍정적 | ✅ 추천 |
| +2→+3 | 200 G | +73% 골드 | 중립 | ⚠️ 신중 |
| +3→+4 | 400 G | +107% 골드 | 부정적 | ❌ 비추천 |
| +4→+5 | 800 G | +149% 골드 | 부정적 | ❌ 비추천 |

**권장 전략:**
- Lv.1-3: 강화 안 함
- Lv.4-5: +1~+2 추천
- Lv.6+: +2~+3 도전
- Lv.8+: +3 이상 도전 (여유 있을 때)

## 14. 부록

### 14.1 용어 정의

- **머지 (Merge)**: 같은 레벨의 아이템 2개를 합쳐 다음 레벨 아이템 1개를 만드는 행위
- **강화 (Enhancement)**: 아이템의 성능을 향상시키는 행위 (레벨은 유지)
- **프레스티지 (Prestige)**: 게임을 초기화하고 영구 보너스를 얻는 시스템
- **리텐션 (Retention)**: 사용자가 게임에 다시 돌아오는 비율
- **DAU (Daily Active Users)**: 일일 활성 사용자 수
- **MAU (Monthly Active Users)**: 월간 활성 사용자 수
